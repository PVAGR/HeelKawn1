# HeelKawn Universe Constitution

Status: Canon Ledger v1.0  
Scope: Binding rules for all simulation, tooling, and AI-authored code.

## Preamble

HeelKawn is a world of witnessed consequence.  
Power is not declared; it is remembered.  
No system may violate causality, continuity, or the primacy of recorded truth.

## Hard Laws (Non-Negotiable)

### LAW 1 — Deterministic Kernel

- Cause equals effect.
- Canon history is deterministic.
- No unseeded randomness may alter historical truth.
- World history must be reproducible from seed + state + ordered inputs.

### LAW 2 — No Chosen Ones

- Players begin as ordinary beings.
- No divine exemption, destiny override, or protected lineage may be assumed.
- Any extraordinary status must emerge from in-world actions and recorded consequence.

### LAW 3 — Memory Over Power

- Actions persist only if witnessed, recorded, or materially traceable.
- `WorldMemory` is the factual spine; derived layers may interpret but must not rewrite.
- Unrecorded claims are non-canonical.

### LAW 4 — Collapse Sequence

- Systemic collapse proceeds in this order:
  1. Trust
  2. Authority
  3. Knowledge
  4. Environment
- Features modeling decline must preserve this ordering unless an explicit canon revision supersedes it.

### LAW 5 — Toolline Primacy

- Foundational progression baseline is:
  1. Hand
  2. Stone
  3. Fire
  4. Knife
- New progression systems must not contradict this root sequence.

## Engineering Compliance Rules

- Never merge a feature that conflicts with any Hard Law.
- If a conflict appears, block implementation and report the exact law violated.
- Derived systems (`WorldMeaning`, UI summaries, narrative overlays) must remain subordinate to factual records.
- Canon updates must be explicit, versioned, and reviewed as constitution changes.

## Enforcement

- This document is a coding contract for humans and autonomous agents.
- In case of ambiguity, prefer the most conservative deterministic interpretation.
